# Read This First

You are working in a repo that follows a standard on-ramp. Start by reading:
- docs/SESSION_CONTEXT.md
- docs/COMMANDS.md
- docs/DECISIONS.md (if present)
- docs/ISSUES.md (if present)

Keep the reply concise. Ask only if something blocks progress.

## Bootstrap Interview (for new projects)
If docs are empty or missing, ask only the minimum questions needed to fill:
- One-sentence project description.
- Current goal for this session.
- 3–5 key files/folders to look at first.
- Known issues/risks.
- How to validate changes.

## Session Wrap (required)
Before ending the session, update:
- docs/SESSION_CONTEXT.md with changes made, current status, and next steps.
- docs/DECISIONS.md if any notable decisions were made.
- docs/ISSUES.md when a meaningful problem was encountered and resolved, including the fix.

## New Project Checklist (from this template)
When starting a brand-new project from this template:
- Create a new repo from template (do not develop directly in `ProjectTemplate`).
- Clone the new repo into its own local folder.
- Rename project-specific labels/placeholders in docs.
- Make an initial commit like `Initialize project from template`.
- Confirm the GitHub remote points to the new repo, not `russfee/ProjectTemplate`.
